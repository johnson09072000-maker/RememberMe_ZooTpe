# RememberMe_ZooTpe ｜ 台北市立動物園智慧導覽與保育互動系統 (PoC)

> **專案定位**：AI 與輕量化 3D 行為模型之智慧動物園保育教育互動系統概念驗證 (Proof of Concept)。

本專案旨在展示如何透過 **輕量化網頁 3D 技術**、**AI 保育知識庫** 與 **行動端 QR Code 掃碼互動**，為台北市立動物園建構一套低導入風險、高擴展性的數位雙生 (Digital Twin) 教育互動引擎。遊客不需下載 App，即可在手機瀏覽器中觀察動物的 24 小時行為節奏，並與 AI 進行即時保育問答。

---

## 🚀 核心功能與展示亮點

1. **純前端 CSS 3D 行為模型**
   * 不依賴外部大型 3D 模型網站（如 Sketchfab），不需下載動輒數十 MB 的 GLB 檔案。
   * 利用純網頁結構與 CSS 3D Transform 打造輕量化動物示意模型，兼顧流暢度與行動端載入速度（正式版可無縫替換為園方授權之 GLB 骨架動畫模型）。
2. **多物種模組化擴張 (Multi-Species Architecture)**
   * 同一套核心互動引擎，已成功驗證支援：**臺灣黑熊**、**無尾熊**、**大象**、**河馬**、**臺灣獼猴**。
   * 每個物種皆具備獨立的 3D 外觀樣式、行為時間軸（巡行、站立、休息）與學習任務。
3. **24 小時行為觀察軸**
   * 將動物的日常習性（覓食、巡行、休息）與時間軸結合，點擊不同時段，3D 模型會同步變更動作，並切換對應的科普教育文案。
4. **AI 保育知識庫問答機制**
   * 模擬遊客與動物的即時 Q&A 互動，答詢內容皆依據專業保育知識庫生成。
5. **保育學習卡與行動延伸**
   * 將參觀體驗轉化為可帶走的數位學習成果，並引導遊客參與園方教育活動、支持動物認養計畫。

---

## 🛡️ 內容安全與合規機制 (Safety Guardrails)

公部門導入 AI 技術時，最重視內容的正確性與合規性。本系統在設計之初即導入以下安全防護網：

* **知識庫範疇限制 (Domain-Specific)**：AI 答詢內容嚴格限制在審核後的保育知識庫內，避免產生 AI 幻覺 (Hallucination)。
* **防範錯誤引導**：不提供任何違反動物福利的互動建議、錯誤飼養觀念，或誤導物種習性的事实。
* **內容前置審查**：系統正式上線前，所有對外生成的科普知識與對話文案，皆可由園方專家進行靜態審核與白名單過濾。
* **教育一致性**：系統不宣稱真實動物具有 AI 意識，完全聚焦於生物學事實與保育宣導。

## 🛠️ 技術堆疊 (Tech Stack)

* **Frontend**: HTML5, CSS3 (Custom Properties, CSS 3D Transforms, Animations), Dynamic JavaScript (Vanilla JS)
* **Design System**: 專屬森林系保育配色（包含台灣黑熊 V 字斑紋、無尾熊樹棲攀爬、大象長鼻等純 CSS 幾何造型實作）
* **Deployment**: GitHub Pages (預期導入)

---

## 📈 低風險 PoC 導入路徑建議

本系統採取**「小步快跑、低阻力驗證」**的策略：
* **零硬體更動**：不需更動既有展區硬體，僅需在現有導覽看板上張貼專屬 QR Code 貼紙。
* **無縫體驗**：遊客掃碼即用，省去 App 安裝的摩擦力。
* **獨立運作**：初期不需串接動物園內部的 IT 系統，資料庫與前端網頁完全獨立託管。
